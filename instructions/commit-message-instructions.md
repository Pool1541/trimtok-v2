Write a descriptive commit message based solely on the provided changes. Follow these strict rules:

- The commit message must be written in English.
- Use conventional commit style.
- Message length is maximum 100 characters.
- The commit message should always be written on a single line.
- Never use lists for the commit message
- Describe what and why, not how.

- Correct examples of conventional commits:
  - feat: add Badge component for displaying styled labels
  - fix: correct typo in user module
  - chore: update dependencies to latest versions
  - refactor: improve code structure in auth module
  - docs: update README with new setup instructions
  - ci: fix CI pipeline configuration

- Incorrect examples of conventional commits:
  - update Badge component for displaying styled labels
  - add pagination to blog module
  - fix issue with login
